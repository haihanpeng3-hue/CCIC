# 一、杯赛题目：基于LoongArch指令集的芯片全流程设计
# 二、参赛组别：仅限A组
# 三、赛题背景

集成电路产业急需具备SoC设计能力的复合型人才，SoC设计强调从系统级角度进行功能划分和模块集成有助于学生建立软硬件协同设计的系统思维，理解从系统规格定义到芯片物理实现的完整技术路径，为后续从事更复杂的芯片设计工作打下基础。本赛题旨在通过基于国产IP的全流程设计，提升国内集成电路产业的自创新能力，减少对外部技术的依赖。
# 四、赛题任务

利用命题企业提供的或自行设计的IP，构建一个基于LoongArch指令集的芯片设计并在FPGA平台上完成原型系统验证，并完成针对芯片电路模块的集成电路后端设计实践，提交设计文档及设计数据，按要求进行答辩演示及现场实操。比赛过程中鼓励参赛队通过软硬件协同优化等方式在系统的功能扩展、性能优化等方面开展创新实践。
本赛题分为初赛、分赛区决赛和全国总决赛三个阶段，各阶段具体要求如下
## （一）初赛阶段

基于杯赛企业提供的IP和芯片设计指导，构建一个基于LoongArch指令集的芯片设计并在FPGA平台上运行指定软件完成测试，提交设计文档。
## （二）分赛区决赛阶段

各参赛队可以在初级赛阶段完成内容基础上，对芯片进行功能扩展和性能优化；同时各参赛队基于集成电路后端设计参考流程完成给定电路模块的后端设计并形成实践报告；最终提交设计文档与数据并进行现场答辩演示。初赛阶段的成绩不计入分赛区决赛。
## （三）全国总决赛阶段

各参赛队可以在分赛区决赛完成内容基础上，进一步对芯片进行功能扩展和性能优化，并根据相应调整内容更新提交的设计文档与数据。
同时各参赛队需进行现场实操并进行答辩演示。现场实操环节将给定数字电路模块，参赛队采用基于标准单元的方式完成指定后端设计步骤。初赛阶段和分赛区决赛阶段的成绩不计入全国总决赛

# 六、参赛资源说明

1. 杯赛企业提供的IP资源包含支持LoongArch架构的处理器核、片上互连总线和部分外设接口。
2. 初赛阶段各参赛队限定使用“计算机系统能力培养远程教学平台”FPGA实验平台，杯赛企业将在整个竞赛期间向杯赛参赛队免费提供远程FPGA实验平台账号。
3. 决赛阶段参赛队允许使用如下FPGA实验平台：
 - 龙芯“CPU设计与体系结构教学实验平台”
 - 龙芯“处理器设计全流程教学设计平台”
4. 比赛中后端设计所涉及的EDA工具、PDK将给出推荐版本，并在比赛期间组织培训进行参考设计流程演示教学。
5. 全国总决赛芯片后端设计的现场实操将使用杯赛企业准备的实验环境，所用的EDA工具、PDK及设计流程与比赛培训中一致。
# 七、其他参赛注意事项

1. 严禁选手提交曾用于过往龙芯中科赛题的作品，不得直接复用已有代码，选手提交作品后将严格和后台数据库进行比对。若本次参赛作品的与过往赛题作品存在相似之处，选手必须随作品附上说明文档，需清晰且明确地标注出本次作品相较于过往相似作品的创新点，以此证明参赛作品在思路、技术应用、解决问题方式等方面具备足够的创新性与独特性，旨在确保竞赛作品的原创价值，维护竞赛的公平公正。若违反要求将会判定为作弊行为，将对参赛团队做出作品不计分、取消参赛资格、通报所在院校等处罚措施。
2. 参赛团队对其独立创作完成的作品享有完整的知识产权；但不包括企业在赛题中所提供的已有技术方案、参考案例、数据库、素材或其他背景资料（以下简称“企业资料”）。对于参加企业命题杯赛的获奖团队作品出题企业就其中由团队原创的部分，享有优先购买权；
3. 大赛组委会及其工作人员应尊重和保护参赛团队的知识产权，组委会及出题企业有权在宣传、展示等场景中使用该参赛作品（含获奖作品）用于赛事推广及相关用途。
4. 如报名本命题参赛即视为该团队同意在大赛结束后对作品进行开源分享。
5. 参赛团队应严格遵守赛题对原创性的要求，如违反要求将取消该队伍作品的参赛资格；
6. 参赛项目可以参考现有公开发表的文献和论文内容，但应当在技术论文和答辩PPT中注明来源，且不能将参考的内容作为自己作品的创新部分。
7. 参赛选手及团队应主动关注大赛官网、公众号、杯赛钉钉群发布的赛事相关的通知，并及时获取、确认相关信息。
